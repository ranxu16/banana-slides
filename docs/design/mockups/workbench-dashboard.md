# 工作台首页设计稿约束

## 页面目标

工作台首页是用户进入系统后的生产入口，不是宣传页。首屏必须直接支持创建汇报、继续项目、查看任务状态和进入常用资源。

## 布局结构

```text
AppShell
  Sidebar
  TopBar
  Main
    PageHeader: 工作台 / 地面光伏事业部 AI 汇报生成平台
    QuickCreateBand
    StatusStrip
    TwoColumnGrid
      RecentProjects
      ExportTasksSummary
    TwoColumnGrid
      CommonTemplates
      SystemStatus / AdminStats
```

## 关键模块

- 快捷创建汇报：一句话生成、从大纲生成、从描述生成、导入文件。
- 最近项目：标题、状态、更新时间、页数、继续编辑、导出入口。
- 常用模板：缩略图、名称、业务场景、使用按钮。
- 导出任务状态：进行中、失败、可下载任务摘要。
- 系统状态：OpenAI/ChatGPT 配置状态、后台任务状态、文件解析服务状态。
- 管理员统计：用户数、项目数、未归属历史项目、失败任务数。

## 视觉约束

- 首屏不使用大 hero。
- 不使用渐变光斑、GitHub 统计、营销卡片。
- 页面背景为浅灰，模块为白色面板，卡片圆角不超过 8px。
- 品牌橙只用于主按钮、选中态和关键状态。
- 信息密度要高，但每个模块标题、操作和状态要清楚。

## 状态要求

- 无项目：显示创建汇报和导入文件入口。
- 有失败任务：显示失败原因入口和重试/查看按钮。
- 配置不可用：显示前往全局配置的操作。

## 验收

- 打开 `/` 第一屏能直接开始创建汇报。
- 旧品牌词不可出现。
- 快捷创建能力不能因重构丢失。
- 1440x900 下无需滚动即可看到创建入口、最近项目和任务摘要。
